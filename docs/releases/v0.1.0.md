# v0.1.0 — Anime Character-First UI Rollout

Release date: 2026-05-20

## Highlights

- Completed a character-first anime UI pass across key web surfaces.
- Replaced remaining runtime-facing octopus glyph usage with `CharacterAvatar` rendering from `/characters/*` assets.
- Unified tentacle emotion mapping so canvas/deck/detail now share consistent status-driven behavior.
- Improved canvas spacing and force-layout tuning for better oracle/tentacle readability.
- Aligned core character avatar and emotion mapping with shipped character asset files.

## Web changes

- `OctopusNode` now uses character rendering for tentacles, removes glyph fallback path, and uses shared emotion context derivation.
- `DeckPrimaryView`, `TentaclePod`, and `CanvasTentaclePanel` now share centralized tentacle emotion mapping.
- `RuntimeStatusStrip`, `ActionCards`, `WorkspaceSetupCard`, `AddTentacleForm`, and `PrimaryViewState` now render `CharacterAvatar` instead of glyphs.
- Added/updated CSS for avatar sizing, glow/emotion styling, and canvas/deck/status integration.
- Improved graph data propagation (`characterId` inheritance for inactive sessions) and increased canvas spread/force parameters.

## Core changes

- Updated built-in character avatar defaults to:
  - `/characters/mika/01-idle.jpg`
  - `/characters/ren/01-idle.jpg`
  - `/characters/yui/01-idle.jpg`
  - `/characters/aki/01-idle.jpg`
- Normalized emotion catalogs and logical state mappings for `ren`, `yui`, and `aki` to match available assets.
- Updated core tests for new emotion/image expectations.
